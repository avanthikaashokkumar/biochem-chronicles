Blood Oxygen and Heart Rate Monitor

A MAX30102-based Arduino project that explores how pulse oximetry sensors estimate heart rate and blood oxygen trends.

Completed Prototype 7 min read Student project note

Materials

  • Arduino UNO R3
  • MAX30102 pulse oximetry sensor module
  • LCD 1602 or OLED display
  • Breadboard
  • Jumper wires
  • USB cable or battery source

Tools

  • Arduino IDE
  • Computer
  • Basic hand tools
  • Multimeter if available

Optional upgrades

  • Bluetooth or Wi-Fi module
  • Data logging
  • Mobile or web dashboard
  • Cleaner enclosure

Project overview

The prototype uses a MAX30102 sensor to detect pulse-related changes in reflected light. An Arduino processes the sensor data and displays estimated heart rate and SpO2 values on a screen.

Status note: This is an educational prototype. It is not clinically validated and should not be used for medical decisions.

Problem being solved

Heart rate and blood oxygen are common vital signs, but the technology behind them can feel like a black box. This project breaks down the sensor, signal, and display process at a beginner level.

Why I built it

I built this because I am interested in biomedical devices and health data. The project helped me connect biology, light-based sensing, and microcontroller programming.

How it works

The sensor shines red and infrared light into a fingertip and measures reflected light. Blood volume changes with each heartbeat, which creates a waveform. The Arduino uses that signal to estimate beats per minute and oxygen saturation trends.

Step-by-step build process

  1. Gather the Arduino, MAX30102 module, display, breadboard, and jumper wires.
  2. Check the module pin labels and voltage requirements before wiring.
  3. Connect the sensor to the Arduino using I2C pins or the pins required by the module.
  4. Connect the display for real-time output.
  5. Install a reliable MAX30102 Arduino library.
  6. Upload a test sketch and check whether the sensor is detected.
  7. Place a fingertip steadily on the sensor and observe the raw signal.
  8. Reduce movement and block strong ambient light during testing.
  9. Compare repeated readings for consistency, not clinical accuracy.
  10. Document what affects the readings.

Calibration or testing

The prototype should be tested for stable contact, repeated readings, and sensitivity to motion. Any comparison with commercial devices should be described as informal unless a proper validation method is used.

Challenges faced

  • Finger movement created noisy readings.
  • Ambient light affected the sensor when the finger was not placed steadily.
  • It was important not to describe the prototype as medically accurate.

What I learned

  • Biomedical data starts as a physical signal.
  • Filtering and stability matter before a reading becomes useful.
  • Honest safety notes are essential for health-related projects.

Future improvements

  • Add simple signal visualization.
  • Add Bluetooth data transfer to a phone or laptop.
  • Log readings over time for learning analysis.
  • Improve the finger placement housing.

Safety or accuracy note

This prototype is for learning and demonstration only. It is not a medical device and should not be used for diagnosis or treatment.
