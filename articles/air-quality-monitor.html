Portable Air Quality Monitor

An Arduino air quality monitoring prototype using DHT11 and MQ sensors to explore gas sensing, calibration, and environmental awareness.

Completed Prototype 7 min read Student project note

Materials

  • Arduino UNO
  • USB cable
  • Breadboard
  • Jumper wires
  • DHT11 sensor
  • MQ-3 sensor
  • MQ-9 sensor
  • MQ-135 sensor
  • LCD or OLED display
  • Optional buzzer or LED alert

Tools

  • Arduino IDE
  • Computer
  • Basic hand tools
  • Multimeter if available

Optional upgrades

  • Bluetooth or Wi-Fi module
  • Data logging
  • Mobile or web dashboard
  • Cleaner enclosure

Project overview

The prototype uses a DHT11 sensor for temperature and humidity and MQ-series sensors for gas-related indicators. The Arduino reads the sensors and can show values or trigger simple alerts.

Status note: This is a low-cost learning prototype. MQ sensor readings need calibration and should not be treated as certified air quality measurements.

Problem being solved

Air quality can change between rooms, roads, and neighborhoods. A portable prototype helps explore how sensors detect changes in the surrounding air, even though it cannot replace certified monitoring systems.

Why I built it

I built this because it connects electronics with public health and environmental science. It also showed me how much calibration matters when sensor readings are affected by real-world conditions.

How it works

MQ sensors change their electrical response when exposed to certain gases. The Arduino reads this response and displays a value or alert. Temperature and humidity readings help provide context because the environment can affect gas sensor behavior.

Step-by-step build process

  1. Gather the Arduino, DHT11, MQ sensors, display, breadboard, and wires.
  2. Read each sensor module label and datasheet because pins and warm-up needs can vary.
  3. Connect and test one sensor at a time before combining them.
  4. Upload simple sketches to read raw values.
  5. Add the display after readings appear correctly in Serial Monitor.
  6. Create rough baseline readings in clean air.
  7. Test changes carefully and avoid unsafe exposure conditions.
  8. Add an LED or buzzer threshold only after baseline testing.
  9. Document how temperature and humidity affect the readings.
  10. Plan better calibration and data logging.

Calibration or testing

Each sensor needs baseline testing and repeated readings. MQ sensors often need warm-up time and can respond to more than one gas, so values should be interpreted carefully.

Challenges faced

  • MQ sensors are not specific enough to identify one pollutant without context.
  • Calibration is more difficult than wiring the circuit.
  • Environmental conditions affected readings.

What I learned

  • Air quality data needs careful interpretation.
  • Sensor arrays can provide a broader picture than one sensor alone.
  • Environmental health projects need honesty about accuracy limits.

Future improvements

  • Add better data logging.
  • Add graphs and trends.
  • Compare readings with local AQI data as a learning exercise.
  • Add a cleaner enclosure and portable power.
  • Explore particulate matter sensors.

Safety or accuracy note

Low-cost sensors need calibration and should be treated as approximate indicators, not certified measurements.
