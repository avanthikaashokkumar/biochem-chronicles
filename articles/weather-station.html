Building My Weather Station

A local weather station using Arduino, a DHT11 sensor, RTC module, and display to track temperature and humidity.

Completed Prototype 6 min read Student project note
ArduinoEnvironmental MonitoringData Science
Download one-page project summary

Materials

  • Arduino UNO
  • DHT11 temperature and humidity sensor
  • RTC module
  • LCD or OLED display
  • Breadboard
  • Jumper wires
  • USB cable or battery source

Tools

  • Arduino IDE
  • Computer
  • Basic hand tools
  • Multimeter if available

Optional upgrades

  • Bluetooth or Wi-Fi module
  • Data logging
  • Mobile or web dashboard
  • Cleaner enclosure

Project overview

The weather station uses a DHT11 sensor to measure temperature and humidity. An RTC module can time-stamp readings, and the Arduino displays the values on a screen.

Status note: This is a student-built weather monitoring prototype. Readings from low-cost sensors are useful for learning but should not replace certified weather instruments.

Problem being solved

Weather apps usually report conditions for a wider area, not the exact temperature and humidity around a desk, balcony, garden, or classroom. A local station gives more immediate environmental context.

Why I built it

I built this because I wanted to see how weather data is collected instead of only reading a forecast. It was also a good way to practice sensors, displays, and simple environmental data collection.

How it works

The DHT11 sensor measures humidity and temperature, sends digital data to the Arduino, and the Arduino updates the display. The RTC helps keep the readings connected to time.

Step-by-step build process

  1. Gather the Arduino, DHT11 sensor, RTC module, display, wires, and breadboard.
  2. Connect the DHT11 sensor according to the module pin labels.
  3. Upload a basic DHT11 test sketch and check Serial Monitor readings.
  4. Connect the display and show temperature and humidity.
  5. Add the RTC module for time-stamped readings if needed.
  6. Place the sensor away from direct sunlight and water.
  7. Compare readings with a known thermometer or hygrometer for a rough check.
  8. Record readings at different times of day.
  9. Improve the layout and protect the circuit.
  10. Plan data logging or graphing as a next step.

Calibration or testing

Readings should be checked against a reliable thermometer or hygrometer when possible. Placement matters because sunlight, body heat, and moisture can change the results.

Challenges faced

  • Sensor placement affected the readings.
  • Different components had to work together without loose connections.
  • The DHT11 is simple and affordable, but not extremely precise.

What I learned

  • Weather data is local and depends heavily on sensor placement.
  • Time-stamping makes environmental data more useful.
  • Simple data collection can lead naturally into data science questions.

Future improvements

  • Add rainfall, wind, or pressure sensors.
  • Save readings to an SD card or cloud dashboard.
  • Create graphs of daily changes.
  • Use a more accurate sensor for comparison.

Safety or accuracy note

Low-cost environmental sensors need calibration and should be treated as approximate indicators, not certified measurements.
